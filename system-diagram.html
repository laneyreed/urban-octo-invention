Learning Mode

A guided, hands-on coding learning system for Claude Code. Three modes, three files, one cross-project memory. This page is a visual reference, see HOW-IT-WORKS.md for the full prose.

1Triggers

What you type to start a session. Each phrase routes to one of the three modes.

Teach Mode

/learning-mode <concept> teach me X I want to learn X explain X with an example how does X work in code

Review Mode

quiz me on what I've learned quiz me from the learning log review what I've learned test me on past concepts review mode

Mental Model Mode

just give me the mental model for X explain X conceptually mental model only I don't want to write code, just explain X

2File architecture

How a trigger flows through the three files that make the system work. Two of them live globally in your home .claude directory, so the same log is available in every project.

You: "/learning-mode lazy init"
User trigger
skills\learning-mode\SKILL.md
Defines tutor behaviour, phases, pacing rules
CLAUDE.md
Global glue, points Claude at the log
learning-log.md (read)
Claude checks what you have already learned
Session runs (phases 1 to 5)
Teach, Review, or Mental Model flow
learning-log.md (append)
New dated entry written at the end

3The three mode flows

Each mode has its own phase sequence. Teach Mode is the heaviest, Mental Model is the lightest.

Teach Mode

Learn a new concept from scratch with explanation, runnable example, and end-of-session integrative quiz.

1. Intake
One calibration question, then sets level (Beginner, Intermediate, Advanced).
1.5. Prerequisite check
Lists up to 3 prerequisite concepts, offers to pivot, embed a refresher, or push through.
2. Example menu
Claude proposes 2 or 3 small examples. You pick one.
3. Quiz check
Asks once whether you want step-level predictive quizzes during the build.
4. Build together
Step by step, explain the concept, flag pattern vs scaffolding, optional quiz gate before running.
4.75. Optional stretch variant
Single harder variant of the same concept, opt-in, skipped if user struggled.
4.5. Integrative quiz
2 or 3 end-to-end questions, tagged solid, shaky, or weak.
5. Wrap-up
Summary, suggested next concept, new log entry appended with review dates.

Review Mode

Quiz yourself on concepts already in your learning log. Spaced-repetition aware.

1. Read the log
Reads learning-log.md. If empty, offers Teach Mode instead.
2. Confirm scope
Defaults to concepts due for review today based on each entry's Next review date.
3. Generate questions
For each concept, generates 2 to 4 predictive questions drawn from the log entry.
4. Run the quiz
One question at a time, with confirmation or correction grounded in the log entry.
5. Wrap-up
Summary of strong vs weak, updates Last reviewed and Next review on each entry.

Mental Model Mode

Concept-only walkthrough with analogy, diagram, and a code sketch. No code written or run.

1. Concept explanation
Definition, analogy, diagram, code sketch. One calibration question, no full intake.
2. Checkpoint questions
1 or 2 predictive questions to confirm the model landed.
3. Offer to escalate
Asks if you want a runnable example. Yes transitions into Phase 2 of Teach Mode.
4. Wrap-up
Shorter log entry tagged Session type: mental-model-only. Tighter spaced-rep intervals.

4Spaced repetition

Review Mode uses a Leitner-style schedule to decide when each concept is due next.

Weak

Got most wrong, or could not explain the why. Next review in 1 day.

Shaky

Got the right answer but with hesitation. Next review in 3 days.

Solid

Got everything right confidently. Doubles each pass, capped at 90 days (30 for mental-model-only).

Solid progression: 7d→ 14d→ 30d→ 60d→ 90d (cap)

5Log entry shape

Every appended entry follows the same shape so it is easy to scan later.

## YYYY-MM-DD, Concept name (project)
Session type: teach | mental-model-only
Concept: One-line definition.
Why it matters: Practical motivation, the bug it solves, or the property it gives you.
The pattern: Minimal code shape worth memorizing.
Real-world reference: Where this pattern shows up in actual code you have written.
Hands-on exercise built: The mini-project you built. Omit for mental-model-only.
Stretch variant: Optional, only if Phase 4.75 ran.
Initial understanding: solid | shaky | weak
Prerequisite caveat: Optional, only if you pushed through a flagged gap.
Last reviewed: YYYY-MM-DD
Next review: YYYY-MM-DD
Next suggested concept: A natural follow-on for the next session.
